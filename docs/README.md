# Documentation Index

Use this folder as the source of truth for deployment and operations.

- `docs/completeness-checklist.md`: criteria and status for "documentation complete as deployed".
- `docs/inventory.md`: deployed resource inventory with ownership and management boundary.
- `docs/deploy-and-operations.md`: bootstrap, deploy, verify, drift checks, rollback guidance.
- `docs/links.md`: project links and ready-to-fill Google Cloud console URLs.
- `docs/runbooks/`: incident-oriented procedures for common failures.
- `docs/archive/`: historical material kept for reference but not authoritative.

If there is a conflict between archived notes and active docs, treat active docs as canonical.
